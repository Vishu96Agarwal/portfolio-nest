01 · About
QA engineer who
thinks like a PM.

Most QA engineers ask "does it work?" I ask "should it work this way?" — and that distinction has saved features from shipping broken, and saved broken features from shipping at all.

My background spans the full product lifecycle: from writing API automation suites in RestAssured + TestNG to sitting in sprint planning and pushing back on acceptance criteria that couldn't be tested. I understand both the code under test and the user story behind it.

I've worked across travel tech, edtech, and enterprise sales platforms, shipping QA ownership that included functional testing, API automation, performance analysis with JMeter, and UAT collaboration with client stakeholders. Currently, I'm deepening my Selenium WebDriver and Page Object Model skills to close the gap toward SDET.

I published a research paper on ML-based sentiment analysis at an IEEE conference in 2024 — because curiosity doesn't stop at the test suite.
